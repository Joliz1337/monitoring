# Remnawave

Интеграция с внешним сервисом Remnawave для сбора статистики по VPN-пользователям и обнаружения аномалий (один HWID = много аккаунтов, географические прыжки и т. д.).

## Что можно делать

- Видеть онлайн-статистику: уникальные пользователи, IP, устройства, ноды
- Смотреть список пользователей и их активность
- Находить HWID-аномалии — устройства, с которых заходят под разными аккаунтами
- Настраивать API-токен Remnawave, интервал сбора, исключения
- Запускать принудительный сбор данных («Collect Now»)

## Ключевые понятия

**HWID** — аппаратный идентификатор устройства, собирается клиентом VPN (MAC, CPU, и т. п.). Один физический девайс = один HWID.

**HWID-аномалия** — когда с одного HWID авторизуются несколько разных пользователей. Признак прокси-фермы, пересылки аккаунта или взлома.

**Collector** — воркер, который периодически опрашивает API Remnawave и складывает данные в БД панели. Без него страница пустая.

**Collect Interval** — как часто collector опрашивает Remnawave. По умолчанию ~60 секунд.

**Online only** — вся статистика (уникальные пользователи, IP, устройства) считается только по **сейчас онлайн** пользователям, не за период.

## Частые вопросы

### Статистика пустая, что проверить?
1. API-токен Remnawave задан и действует
2. Collector запущен (статус «running» в настройках)
3. URL Remnawave доступен из панели (сетевая доступность)
4. В Remnawave реально есть онлайн пользователи

### HWID-аномалия — это точно прокси-ферма?
Не всегда. Варианты:
- Человек пересылает подписку друзьям (бытовое нарушение)
- Пересобрал ОС, HWID изменился, старые записи висят
- Виртуалка клонировалась — HWID одинаковый, а пользователи разные
- Реальная прокси-ферма

Смотрите по контексту: количество аккаунтов на HWID, частота подключений, страны.

### Collect Now — что делает?
Запускает принудительный цикл опроса Remnawave прямо сейчас, не ждёт следующий интервал. Полезно после изменения настроек на Remnawave, чтобы сразу увидеть новые данные.

### Как исключить тестовых пользователей?
В настройках добавьте UUID/HWID в exclusion list. Эти идентификаторы collector будет пропускать и не генерировать по ним аномалии.

### Что нужно для работы интеграции?
- Развёрнутый Remnawave с включённым API
- API-токен с правами чтения
- Сетевая связность панели → Remnawave
- Заполненные настройки на вкладке Settings этой страницы

## Если что-то не работает

- Collector в статусе `stopped` → перезапустите через панель, смотрите логи воркера
- HWID-аномалии пустые → нужно хотя бы 10+ минут сбора данных
- API не отвечает → проверьте URL и токен в настройках
