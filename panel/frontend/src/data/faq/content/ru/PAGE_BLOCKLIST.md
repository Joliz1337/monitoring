# Блок-лист

Централизованная IP-фильтрация: чёрные списки, белый список, глобальные и per-server правила, внешние источники. На нодах работает через ipset.

## Что делает
- **Global Rule** — блокировка применяется на все серверы автоматически.
- **Per-Server Override** — индивидуальное правило для сервера, добавляется поверх глобальных.
- **IN** блокирует входящие подключения к серверу, **OUT** — исходящие соединения сервера.
- **Source** — внешний URL со списком IP/CIDR, подтягивается фоном раз в час (или кнопкой Refresh).
- **Белый список (Allowlist)** — доверенные IP/CIDR, которым доступ разрешён всегда.

## Полезно знать
- Белый список приоритетнее: на ноде `ACCEPT` стоит выше всех `DROP`, поэтому доверенный IP пройдёт, даже если попадает под заблокированный CIDR.
- `192.168.0.0/24` = 256 адресов, `/32` = один IP. Чем меньше число после слэша, тем больше адресов.
- Blocklist банит весь трафик (ipset/netfilter), Firewall HAProxy — только порты HAProxy.
- Источник в статусе `error` — нажмите Refresh и смотрите ошибку: DNS, HTTP 403/404, таймаут или неверный формат.
- ipset спокойно держит миллионы IP. Проверить применение на ноде: `ipset list`.
