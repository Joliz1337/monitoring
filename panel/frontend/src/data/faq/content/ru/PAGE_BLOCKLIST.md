# Блок-лист

Централизованное управление IP-фильтрацией: чёрные списки, глобальные и per-server правила, импорт внешних источников. Используется через ipset на нодах.

## Что можно делать

- Создавать глобальные правила блокировки для входящего (IN) и исходящего (OUT) трафика
- Переопределять глобальные правила на уровне отдельных серверов
- Импортировать готовые списки из внешних источников (Spamhaus, FireHOL, свои URL)
- Смотреть статус синхронизации источников
- Ручной refresh источника в любой момент

## Ключевые понятия

**IN (inbound)** — блокировка входящих подключений. Никто с этих IP не сможет подключиться к вашему серверу.

**OUT (outbound)** — блокировка исходящих подключений. Ваш сервер не сможет открыть соединение к этим IP.

**CIDR-блок** — нотация подсети. `192.168.0.0/24` = 256 адресов (от .0 до .255). `/32` = один IP. Чем меньше число после слэша, тем больше адресов.

**Global Rule** — правило применяется на **все** серверы автоматически.

**Per-Server Override** — индивидуальное правило для конкретного сервера. Дополняет или переопределяет глобальные.

**Source** — внешний URL со списком IP/подсетей, который периодически подтягивается. Типичные источники: Spamhaus DROP, Team Cymru, FireHOL.

**ipset** — механизм ядра Linux, через который блокировка работает эффективно даже для миллионов IP. Namespace-и ipset заполняются агентом ноды по данным с панели.

## Частые вопросы

### Глобальное правило vs серверное — порядок применения?
Работают оба одновременно. Если IP заблокирован глобально — он заблокирован везде. Если есть индивидуальное правило для сервера — оно добавляется сверху к глобальным, не отменяя их.

### Чем отличается Blocklist от Firewall HAProxy?
**Blocklist** — фильтрация на уровне ядра (ipset/netfilter), работает для всего трафика. **Firewall HAProxy** — правила на уровне HAProxy, только для его портов. Blocklist — это «ban everywhere», HAProxy firewall — «ban this service».

### Источник показывает `error`, что делать?
Нажмите Refresh, посмотрите конкретную ошибку: DNS не резолвится, HTTP 403/404, таймаут, неверный формат. Типично — URL устарел или требует аутентификации.

### Как часто обновляются источники?
По умолчанию раз в час фоновый процесс. Можно принудительно кнопкой Refresh.

### Можно ли добавить свой IP в whitelist?
Whitelist не поддерживается — только blocklist. Если нужно гарантировать доступ, удостоверьтесь, что вашего IP нет в блок-листе (нет в глобальных правилах, нет в серверных, не пришёл из внешних источников).

## Если что-то не работает

- Блокировка не применяется → проверьте, что нода получила обновление (лог агента) и ipset реально обновлён: `ipset list`
- Не могу удалить источник → сначала остановите автосинхронизацию, потом удалите
- Внешний источник не парсится → проверьте формат (должны быть IP или CIDR по одному на строку, без комментариев)
