# Режим NIC: Multiqueue, Hybrid или RPS

Три способа раскидать обработку сетевых пакетов по ядрам CPU. По умолчанию все пакеты валятся на одно ядро, и оно становится узким местом.

## Режимы
- **Multiqueue (аппаратный)** — карта раздаёт пакеты по очередям, каждая на своё ядро. Максимум скорости, минимум overhead. Задействует столько ядер, сколько у NIC очередей.
- **Hybrid (аппаратный + RPS)** — нужен, когда очередей у NIC меньше, чем ядер CPU (типично для consumer-железа: Intel I225-V, I226-V, I219). IRQ на первые N ядер, RPS размазывает остальное — работают все ядра.
- **RPS (программный)** — если карта не умеет очереди вовсе, ядро само распределяет пакеты. Медленнее аппаратных режимов, но работает везде.

## Как выбрать
Смотрите в карточке ноды поле «Hardware Multiqueue».
- HW-очередей `>=` ядер → **Multiqueue**
- HW-очередей `<` ядер → **Hybrid** (панель подсветит бейдж «рекомендуется»)
- `not supported` → только **RPS**

## Полезно знать
- Применение идёт на лету через `ethtool` и sysfs, соединения не рвутся; при смене режима остальные удаляются автоматически.
- Помогло или нет видно по `mpstat -P ALL 1`: до — одно ядро 100% на `si`, после — нагрузка ровно по всем ядрам.
- Multiqueue применился, но загрузка не изменилась — проверьте `ls /sys/class/net/eth0/queues/`, там должно быть несколько `rx-*` и `tx-*`.
