# Обновления

Версии панели и всех нод в одном месте. Видно, что установлено и что доступно, обновление — одной кнопкой.

## Что делает
- Показывает текущую и последнюю доступную версию панели
- Обновляет панель одной кнопкой (с перезапуском)
- Показывает все ноды с их версиями и статусом
- Обновляет отдельную ноду или все сразу
- Проверяет наличие обновлений принудительно (Refresh)

## Полезно знать
- Статусы: **Up To Date** — обновлять не нужно; **Update Available** — есть свежее; **In Progress** — идёт обновление (~30–60 сек); **Offline** — нода недоступна, сначала верните в онлайн.
- Обновление панели запускает миграции БД и перезапуск: веб-интерфейс недоступен 10–30 сек, ноды продолжают работать.
- Обновление ноды трогает только агент в `/opt/monitoring/node/`, метрики не теряются, занимает ~20 сек.
- Offline-ноду через сеть не обновить — почините связь, либо вручную на сервере через `apply-update.sh`.
- Отката к старой версии из панели нет — нужный тег ставится вручную через установщик.
- Refresh не видит новую версию — обычно нет интернета на панели или лимиты GitHub API.
