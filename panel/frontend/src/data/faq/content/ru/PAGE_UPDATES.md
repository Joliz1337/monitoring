# Обновления

Управление версиями панели и всех нод. Видно, какая версия установлена, какая доступна, и здесь же можно обновиться одной кнопкой.

## Что можно делать

- Видеть текущую версию панели и последнюю доступную версию
- Обновлять панель одной кнопкой (панель перезапустится)
- Видеть список всех нод с их текущими версиями и статусом
- Обновлять отдельную ноду или все сразу
- Проверять наличие обновлений принудительно (Refresh)

## Ключевые понятия

**Panel Version** — версия веб-панели (backend + frontend).

**Node Version** — версия агента на конкретном сервере. Каждая нода обновляется независимо.

**Latest Version** — последняя доступная версия из репозитория обновлений.

**Up To Date** — установленная версия совпадает с последней. Обновлять не нужно.

**Update Available** — есть более свежая версия. Обновление рекомендовано.

**Offline** — нода не отвечает, обновить нельзя. Нужно сначала вернуть в онлайн.

**In Progress** — обновление запущено, идёт скачивание и применение. Обычно 30–60 секунд.

## Частые вопросы

### Что произойдёт при обновлении панели?
Панель скачает новую версию, запустит миграции БД (если есть), и перезапустится. Во время перезапуска веб-интерфейс недоступен 10–30 секунд. Ноды продолжат работать — они не зависят от панели в реальном времени.

### Нода offline, как обновить?
Никак — через сеть агент недоступен. Варианты:
1. Вернуть ноду в онлайн (починить сеть/сервис), потом обновить как обычно
2. Зайти на ноду по SSH и обновить вручную через `apply-update.sh`
3. Пересоздать ноду с нуля из актуального установщика

### Обновление ноды — что меняется?
Обновляется только агент (Python-сервис), метрики не теряются. Переустанавливает/обновляет файлы в `/opt/monitoring/node/`, перезапускает systemd-сервис. Занимает ~20 секунд.

### Можно ли откатиться к старой версии?
Напрямую — нет. Если обновление что-то сломало, нужно вручную ставить предыдущую версию через установщик с указанием тега версии.

### Почему кнопка «Refresh» не находит новую версию, хотя она есть?
Панель проверяет репозиторий по HTTPS. Возможные причины:
- Нет интернета на панели
- GitHub API лимиты
- Изменился URL репозитория (редко)

## Если что-то не работает

- Обновление «зависло» → смотрите логи панели (`journalctl -u monitoring-panel`), обычно застревает на миграциях БД
- Ошибка при обновлении ноды → смотрите статус карточки, там должно быть сообщение; чаще всего диск заполнен
- Версия показывается «Unknown» → агент не смог сообщить версию, проверьте связь
