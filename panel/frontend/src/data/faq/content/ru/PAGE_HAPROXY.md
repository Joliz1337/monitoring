# HAProxy

Балансировщик и reverse-proxy на одном сервере. На странице — статус сервиса, правила балансировки, SSL-сертификаты и firewall.

## Что показывает
- Статус HAProxy (running/stopped) и валидность конфига
- Правила балансировки: какой порт куда отправляет трафик
- SSL-сертификаты: выпуск через Let's Encrypt, ручная загрузка, автообновление
- Firewall: какие порты, протоколы и IP разрешены
- Сырой `haproxy.cfg` в модалке — для опытных

## Полезно знать
- **Frontend** — порт, который слушает HAProxy (80, 443, 8080), сюда подключаются клиенты. **Backend** — пул реальных серверов, между которыми идёт распределение.
- **Reload** перечитывает конфиг без обрыва соединений, дёргать можно часто. **Restart** обрывает всё — только если HAProxy завис.
- Перед применением конфиг проверяется (`haproxy -c`). Если невалиден — reload не пройдёт, чтобы сервис не лёг.
- «HAProxy stopped» — почти всегда битый конфиг, причина видна в Config Validation.
- Сертификат не генерируется — проверьте, что домен указывает на этот сервер и порт 80 открыт для ACME.
- Правило не применилось — нажмите Reload.
