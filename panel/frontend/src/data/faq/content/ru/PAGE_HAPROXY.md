# HAProxy

Управление балансировщиком нагрузки и reverse-proxy HAProxy на конкретном сервере. Страница объединяет статус сервиса, правила балансировки, SSL-сертификаты и firewall.

## Что можно делать

- Видеть текущий статус HAProxy (running/stopped) и валидность конфига
- Создавать, редактировать, удалять правила балансировки
- Управлять SSL-сертификатами: генерация через Let's Encrypt, ручная загрузка, автообновление
- Настраивать firewall: разрешать/запрещать порты, протоколы, IP
- Редактировать сырой `haproxy.cfg` в модалке (для опытных)
- Перезагрузить HAProxy после изменений кнопкой «Reload»

## Ключевые понятия

**Frontend** — то, что слушает HAProxy. Обычно порт: 80, 443, 8080. Клиенты подключаются сюда.

**Backend** — список реальных серверов, между которыми HAProxy распределяет запросы. Один frontend → один backend пул из N серверов.

**Reload** — перечитать конфиг без разрыва текущих соединений. Безопасно, можно дёргать часто. В отличие от **restart**, который обрывает всё.

**Config Validation** — проверка синтаксиса конфига перед применением. Если невалиден — HAProxy откажется перезагружаться, чтобы не «лечь» целиком.

**SSL-сертификат** — файл с ключом и цепочкой, которым HAProxy шифрует HTTPS. Имеет срок действия (обычно 90 дней у Let's Encrypt).

## Частые вопросы

### Как добавить HTTPS на мой домен?
1. Откройте раздел SSL сертификатов
2. Нажмите «Generate» (Let's Encrypt) или «Upload» (свой сертификат)
3. Для Let's Encrypt укажите email и домен, панель сама выполнит ACME challenge
4. После выпуска добавьте ссылку на сертификат в правило HAProxy, перезагрузите

### Что быстрее — Reload или Restart?
**Reload** — секунды, без обрыва текущих соединений. **Restart** — обрывает всё, нужен только если HAProxy завис и Reload не помогает.

### Сертификат скоро истечёт — что делать?
Нажмите «Renew» рядом с сертификатом. Для Let's Encrypt работает автообновление (если настроено в Wildcard SSL). Для ручных сертификатов — загрузите новый файл.

### Конфиг невалидный, как понять где ошибка?
В разделе статуса HAProxy показывается вывод `haproxy -c -f /etc/haproxy/haproxy.cfg`. Там точная строка и причина: неизвестная директива, опечатка в backend, отсутствующий сертификат.

### Firewall HAProxy vs Blocklist vs ipset — это одно и то же?
Нет. Firewall HAProxy — правила на уровне iptables/nftables для портов HAProxy. Blocklist — глобальный список заблокированных IP на уровне ядра. ipset — механизм, через который Blocklist и TorrentBlocker добавляют IP. Работают независимо.

## Если что-то не работает

- «HAProxy stopped» → посмотрите причину в Config Validation, скорее всего битый конфиг
- Сертификат не генерируется → проверьте, что домен направлен на этот сервер и порт 80 открыт для ACME challenge
- Правило не применяется → не забыли нажать «Reload»?
