# Триггер: Сервер offline

Самый важный алерт: если нода перестаёт отвечать на проверки связи, вы узнаёте первыми. Когда сервер возвращается — приходит уведомление «back online».

## Параметры
- **Fail threshold** — сколько неудачных проверок подряд до объявления offline. Меньше = быстрее реакция, но больше ложных срабатываний. Обычно 3–5.
- **Check interval** — как часто проверяется доступность (из общих настроек мониторинга).
- **Cooldown** — пауза между повторными алертами.

## Полезно знать
- Панель пингует API агента ноды; offline объявляется после `fail_threshold` неудач подряд.
- При `check_interval=30s` и `fail_threshold=10` алерт придёт только через 5 минут — слишком долго.
- Recovery-алерт «вернулся онлайн» включён по умолчанию.
- Сервер залип в offline, хотя доступен — проверьте вручную «Тест» на странице Servers.
