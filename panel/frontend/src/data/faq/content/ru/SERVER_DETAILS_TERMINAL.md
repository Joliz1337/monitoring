# Встроенный терминал

Выполнение команд на сервере без захода по SSH. Удобно для быстрых проверок и не требует отдельного SSH-доступа.

## Что можно делать

- Выполнять одну команду (Exec mode)
- Запускать многострочный скрипт (Script mode)
- Выбирать shell: `sh` (совместимый) или `bash` (расширения)
- Настраивать timeout (30 сек — 10 мин)
- Видеть полный вывод: stdout, stderr, exit code
- История команд (сохраняется в браузере)

## Ключевые понятия

**Exec mode** — одна команда, выполняется в текущем shell.

**Script mode** — несколько строк, сохраняются во временный файл и выполняются как скрипт. Поддерживает переменные, циклы, функции.

**Timeout** — лимит времени. По истечении процесс получает SIGTERM, потом SIGKILL.

**Exit code** — код возврата. `0` = успех, `1–255` = ошибка.

## Типичные команды

### Проверки
```bash
uptime                      # время работы + load average
df -h                       # диск
free -m                     # память
who                         # кто залогинен
last -20                    # кто заходил
```

### Процессы
```bash
ps auxf | head -30          # топ процессов
pgrep -l nginx              # найти по имени
kill 1234                   # завершить процесс
kill -9 1234                # жёстко
```

### Сеть
```bash
ss -tlnp                    # слушающие порты
ip addr                     # интерфейсы
ping -c 3 8.8.8.8           # проверка связи
curl -I https://example.com # HTTP заголовки
```

### Сервисы
```bash
systemctl status nginx      # статус
systemctl restart nginx     # перезапуск
journalctl -u nginx -n 50   # логи
```

## Частые вопросы

### Нужен ли sudo?
Нет. Агент работает от root, sudo не нужен.

### Интерактивные команды?
Не поддерживаются. `htop`, `top`, `vim`, `passwd`, `nano`, `less`, `more` — не работают. Используйте неинтерактивные аналоги:
- вместо `less` → `cat` или `head`/`tail`
- вместо `vim` → `sed` или `echo > file`
- вместо `top` → `ps`

### Почему вывод обрезан?
Очень большой вывод (MB+) может быть обрезан для производительности. Пишите команды, которые возвращают компактный результат: `| head -100`, `| tail -50`, `| wc -l`.

### История команд сохраняется?
Да, в localStorage браузера. Только для вас, не шарится с другими администраторами.

### Можно ли отменить команду?
Частично. Если команда ещё выполняется (не прошло timeout) — перезагрузить страницу. Команда продолжит работать на сервере, но результат вы не увидите. Через SSH можно было бы сделать `kill`, но здесь нет.

## Если что-то не работает

- Команда зависла → увеличьте timeout или попробуйте `timeout 10 your-command`
- Exit code 127 → команда не найдена; полный путь (`/usr/bin/cmd`) или пакет не установлен
- Вывод пустой, exit 0 → команда не печатала ничего (например, `touch` — это нормально)
