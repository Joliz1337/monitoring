# Wildcard SSL

Управление wildcard-сертификатом (например, `*.example.com`), его выпуском, обновлением и раскаткой на несколько серверов. Работает через Let's Encrypt + Cloudflare DNS challenge.

## Что можно делать

- Выпускать новый wildcard-сертификат через Let's Encrypt
- Обновлять сертификат вручную или автоматически перед истечением
- Разворачивать сертификат на несколько серверов с указанием пути и команды reload
- Настраивать Cloudflare API для автоматизации DNS challenge
- Видеть дату истечения и days-left каждого сертификата

## Ключевые понятия

**Wildcard SSL** — сертификат, покрывающий основной домен и все поддомены (`*.example.com` → `api.example.com`, `www.example.com`, и т. д.). Один сертификат = все поддомены, удобно.

**ACME** — стандартный протокол автоматического выпуска SSL-сертификатов. Let's Encrypt работает через ACME.

**DNS challenge** — способ доказать Let's Encrypt, что вы владеете доменом: нужно создать TXT-запись в DNS. Для wildcard это единственный вариант (HTTP challenge не поддерживается).

**Cloudflare API Token** — ключ, с которым панель сама создаёт и удаляет TXT-записи в DNS Cloudflare. Без него придётся создавать записи вручную.

**Deploy Path** — куда на сервере класть файлы сертификата. Обычно `/etc/letsencrypt/live/<domain>/`. Там будут `fullchain.pem` и `privkey.pem`.

**Reload Cmd** — команда, которая перезапускает веб-сервер после обновления сертификата. Обычно `systemctl reload nginx` или `systemctl reload haproxy`.

**Renew Days Before** — за сколько дней до истечения запускать автообновление. По умолчанию 30.

## Частые вопросы

### Зачем wildcard, если можно по сертификату на каждый поддомен?
Один wildcard покрывает любой поддомен, включая новые, которые вы добавите потом. Не нужно каждый раз запускать ACME.

### Почему именно Cloudflare, а не другой провайдер?
Используется DNS challenge, и Cloudflare даёт простой API с маленьким пермишеном (только Zone.Edit). Другие провайдеры тоже можно, но под них нужен отдельный код.

### Что такое «Reload cmd» и почему он важен?
Веб-сервер загружает сертификат при старте и держит в памяти. После обновления файлов нужно сказать ему «перечитай». Без этого новые файлы лежат, но сервер использует старый (пока не перезапустится).

### `days left` красный — что делать?
Если отрицательный — сертификат уже истёк, клиенты получают ошибки SSL. Срочно нажать Renew или проверить, почему автообновление не сработало (чаще всего Cloudflare API Token истёк).

### Как создать Cloudflare API Token?
На сайте Cloudflare → My Profile → API Tokens → Create Token. Права: `Zone.Zone.Read`, `Zone.DNS.Edit` на нужную зону. Скопировать и вставить в панель.

## Если что-то не работает

- Ошибка при выпуске → проверьте Cloudflare Token, зону, права
- Deploy работает, но сервер всё ещё шлёт старый сертификат → Reload Cmd не выполнен или неверен
- Auto-renew не срабатывает → проверьте, что `Auto Renew` включён в настройках
