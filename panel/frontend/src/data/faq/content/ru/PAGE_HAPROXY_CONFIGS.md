# Конфигурации HAProxy

Глобальные шаблоны HAProxy: одна конфигурация на много серверов. Удобно, когда нужно одинаковое поведение на 10+ нодах без ручной правки каждой.

## Что делает
- Профиль — набор правил балансировки, который вы явно синхронизируете на выбранные серверы
- Правило: слушать порт → отправить на один target или балансировать между пулом backend
- Алгоритм балансировки: roundrobin, leastconn, source, first
- Sticky sessions — привязка клиента к одному backend
- Health checks (TCP или HTTP) для проверки живости backend
- Сырой конфиг вручную — для опытных

## Полезно знать
- **roundrobin** — по кругу, для коротких HTTP-запросов. **leastconn** — на backend с меньшим числом соединений, для долгих (WebSocket, БД). **source** — по хешу IP клиента, дешёвая «прилипаемость» без cookie.
- **Backup server** включается только когда основные упали.
- **Send PROXY** передаёт реальный IP клиента — нужно, если за HAProxy стоит nginx/приложение, которому важен client IP.
- Статусы: **synced** — применён, **pending** — в процессе, **failed** — ошибка (битый конфиг или сервер offline).
- Health check показывает DOWN при живом сервере — проверьте порт, firewall и что `/health` отвечает 200.
- Катите на прод по одному серверу: сначала тестовый → проверка → весь пул.
