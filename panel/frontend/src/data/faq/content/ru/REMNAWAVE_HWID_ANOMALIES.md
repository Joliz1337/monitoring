# HWID-аномалии

Ситуации, когда один физический девайс (HWID) используется несколькими аккаунтами — или один аккаунт прыгает между разными HWID. Признак нарушения правил использования, ферм аккаунтов, пересылки подписок.

## Что такое HWID

Hardware ID — отпечаток устройства, собираемый клиентом VPN. Включает MAC, CPU, BIOS, GPU и другие параметры. Одно физическое устройство = один стабильный HWID. Смена ОС или обновление железа может изменить HWID.

## Типы аномалий

**One HWID → multiple users** — одно устройство, разные пользователи. Варианты:
- Пересылка аккаунта друзьям/знакомым
- Прокси-ферма: один VPS, много аккаунтов
- Клонированная VM (один образ, разные аккаунты)

**One user → multiple HWID** — один пользователь, много устройств за короткое время. Варианты:
- Легальный: телефон + ноутбук + десктоп
- Подозрительный: 20 устройств за час
- Ферма пользователей, каждый с одним устройством

**Geo-jump** — устройство за час «переместилось» из одной страны в другую. Физически невозможно без VPN внутри VPN. Часто — прокси-ферма использует несколько выходных точек.

## Severity

- **Low** — 2–3 аккаунта на HWID, редкие переключения. Возможно бытовое нарушение.
- **Medium** — 5+ аккаунтов на HWID. Активная пересылка или ферма.
- **High** — 20+ аккаунтов, быстрые переключения, geo-jumps. Почти наверняка автоматизация.

## Частые вопросы

### Severity High = точно ферма?
С высокой вероятностью, но нужно смотреть контекст. Может быть семья с общим компьютером (все зашли в один день). High — повод разобраться, не повод сразу банить.

### Как реагировать?
Встроенной «реакции» нет — только отображение. Решения принимаете вы на своей стороне: предупреждение, ограничение, бан.

### Сбрасываются ли аномалии?
По мере «забывания» — старые записи уходят из статистики. Если поведение прекращается, severity со временем падает.

### Что такое «HWID changed»?
Клиент с тем же UUID вдруг присылает другой HWID. Варианты:
- Переустановка ОС
- Замена оборудования
- Использование на другом устройстве

Само по себе не страшно, но частые смены + много разных HWID = подозрительно.

## Если что-то не работает

- Аномалии не обновляются → проверьте, что collector работает
- Список пустой → реальных аномалий нет, либо collector только что запустился
- Количество завышено → есть ложные срабатывания (общий компьютер в семье), их можно игнорировать
