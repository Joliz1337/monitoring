# Массовые операции

Выполнение одного действия сразу на нескольких серверах. Экономит время, когда надо перезапустить HAProxy на 50 нодах, добавить firewall-правило, или выполнить команду через shell.

## Что можно делать

- Выбирать серверы из списка (по одному, по папке, все сразу), искать по имени/IP
- Управлять сервисом HAProxy: start / stop / restart на всех выбранных
- Добавлять и удалять отслеживаемые порты трафика массово
- Создавать и удалять firewall-правила на группе серверов
- Выполнять произвольные shell-команды через встроенный терминал
- Смотреть результаты по каждому серверу: успех/ошибка, полный вывод

## Ключевые понятия

**Bulk action** — одна операция, применяемая к множеству серверов параллельно. Результат собирается по каждому серверу отдельно.

**Shell timeout** — сколько секунд ждать выполнение команды на одном сервере. Превышение → команда прерывается (SIGTERM).

**Partial success** — часть серверов выполнила успешно, часть — нет. Нормальная ситуация: один сервер offline, один timeout, остальные ок.

**Scope (выбор)** — можно выбрать серверы галочками, либо выбрать всю папку — тогда любое действие применится ко всем, что в ней.

**Exec mode vs Script mode** — в терминале: Exec — одна команда; Script — многострочный скрипт, сохраняется и выполняется как единый блок через shell.

## Частые вопросы

### Можно ли отменить запущенную массовую операцию?
Нет. Операция отправляется параллельно на все серверы и дальше уже не контролируется. Перед запуском дважды проверьте выбор серверов, особенно для опасных команд (`rm`, `iptables -F`, `reboot`).

### Почему на одном сервере успех, а на другом timeout?
Разная нагрузка, сеть, агент может быть перегружен. Увеличьте timeout для следующей попытки. Типичные причины: медленный диск, много процессов, нестабильная сеть.

### Shell через bulk vs индивидуальный терминал — разница?
Функционально одно и то же. Bulk удобен, когда нужно одно и то же на многих серверах. Индивидуальный терминал — когда работаете с одним сервером и следите за выводом в реальном времени.

### Как выбрать все серверы из папки «Production»?
Кликните галочкой на самой папке в левой части страницы — выделятся все серверы внутри. Можно комбинировать несколько папок.

### Куда деваются результаты после закрытия страницы?
Результаты живут в памяти страницы и пропадают при перезагрузке. Логи команд — в БД панели, но история массовых запусков не сохраняется.

## Если что-то не работает

- Некоторые серверы в «pending» навсегда → агент упал или сеть легла, повторите операцию по выжившим
- Команда зависла → увеличьте timeout или убедитесь, что команда не ждёт интерактивный ввод
- `sudo` не работает → агент уже запущен от root (по умолчанию), `sudo` не нужен
