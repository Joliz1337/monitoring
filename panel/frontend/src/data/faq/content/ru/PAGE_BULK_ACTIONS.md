# Массовые операции

Одно действие сразу на нескольких серверах. Экономит время, когда нужно перезапустить HAProxy на 50 нодах или раскатать команду на группу.

## Что делает
- Выбор серверов галочками, по папке или все сразу, поиск по имени/IP.
- HAProxy: start / stop / restart на выбранных.
- Массовое добавление/удаление отслеживаемых портов и firewall-правил.
- Произвольные shell-команды через терминал, результат по каждому серверу отдельно.

## Полезно знать
- Выбрать всю папку — галочка на самой папке слева, можно комбинировать несколько.
- Отменить запущенную операцию нельзя — перед запуском проверьте выбор, особенно для `rm`, `iptables -F`, `reboot`: ошибка повторится на всех.
- **Partial success** — норма: один сервер offline, один по timeout, остальные ок. Повторите по выжившим.
- Разный результат на серверах — разная нагрузка/сеть, увеличьте timeout (превышение → SIGTERM).
- `sudo` не нужен — агент работает от root.
- Результаты живут в памяти страницы и пропадают при перезагрузке.
