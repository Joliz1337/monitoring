# SSH-ключи

Загрузка и управление публичными SSH-ключами для авторизации без пароля. Безопаснее любого пароля и удобнее в использовании.

## Как загрузить

1. На вашей машине: `ssh-keygen -t ed25519 -C "my@email"` (если ключа ещё нет)
2. Посмотреть публичный: `cat ~/.ssh/id_ed25519.pub`
3. Скопировать всю строку (начинается с `ssh-ed25519 ...`)
4. В панели: нажать «Add Key», вставить, задать имя
5. Выбрать серверы для применения, нажать Apply
6. Ключ добавится в `~/.ssh/authorized_keys` нужного пользователя

## Ключевые понятия

**Publickey** — то, что можно раздавать. Содержит математическую «проверку» вашей секретной части.

**Private key** (`id_ed25519` без `.pub`) — остаётся **только у вас**, никуда не загружается. Если утечёт — ваши ключи скомпрометированы.

**Fingerprint** — хеш публичного ключа, удобен для опознания. Одинаковый fingerprint = один и тот же ключ.

**authorized_keys** — файл на сервере в `~/.ssh/authorized_keys`, содержит список разрешённых публичных ключей. Один ключ = одна строка.

## Какой тип ключа выбрать

- **ed25519** — современный, быстрый, короткий. Рекомендуется.
- **rsa 4096** — старый стандарт, совместим со всем. Используйте, если ed25519 не поддерживается.
- **dsa, ecdsa** — не рекомендуется, устаревшие или проблемные.

Команда: `ssh-keygen -t ed25519 -C "your@email"`.

## Частые вопросы

### Нужен ли passphrase на приватный ключ?
Да, очень рекомендуется. Без passphrase утечка приватника = мгновенный взлом. С passphrase атакующему нужно ещё его знать/перебрать. Потеряете passphrase — ключ бесполезен, создавайте новый.

### Как пользоваться ключом с passphrase без постоянного ввода?
На Linux/Mac — `ssh-agent`. Один раз ввели passphrase после логина — забыли. `ssh-add ~/.ssh/id_ed25519`.

### Загрузил ключ, но всё равно просит пароль
Частые причины:
1. Права на `~/.ssh/` должны быть 700 и `authorized_keys` — 600. Иначе sshd игнорирует.
2. `PubkeyAuthentication yes` в sshd_config
3. SELinux блокирует чтение (редко)
4. Неверный ключ (опечатка при копипасте)

Запустите `ssh -v user@server` и смотрите, почему ключ не принят.

### Можно ли удалить ключ удалённо?
Да, через эту же страницу: удалите запись, Apply — агент перепишет `authorized_keys` на сервере.

## Если что-то не работает

- Ключ добавлен, но вход требует пароль → проверьте права файлов `~/.ssh/`
- Ключ не применяется → проверьте связь с нодой
- Удалить ключ не получается → возможно, пользователь, для которого ключ, удалён на сервере
