# SSH-ключи

Загрузка публичных ключей для входа без пароля. Безопаснее пароля и удобнее: ключ нельзя перебрать за разумное время.

## Как загрузить
1. Создать ключ (если нет): `ssh-keygen -t ed25519 -C "my@email"`
2. Посмотреть публичный: `cat ~/.ssh/id_ed25519.pub`
3. Скопировать всю строку (начинается с `ssh-ed25519 ...`)
4. В панели «Add Key» — вставить, задать имя
5. Выбрать серверы, нажать Apply — ключ попадёт в `~/.ssh/authorized_keys`

## Полезно знать
- Тип ключа: **ed25519** — современный, рекомендуется; **rsa 4096** — если ed25519 не поддерживается; dsa/ecdsa — не использовать.
- Приватный ключ (`id_ed25519` без `.pub`) никуда не загружается и остаётся только у вас. Утечёт — ключ скомпрометирован.
- Passphrase на приватный ключ очень желателен; чтобы не вводить каждый раз — `ssh-add ~/.ssh/id_ed25519` через `ssh-agent`.
- Вход всё равно просит пароль — проверьте права: `~/.ssh/` должно быть `700`, `authorized_keys` — `600`, иначе sshd ключ игнорирует. Диагностика: `ssh -v user@server`.
- Удалить ключ удалённо можно с этой же страницы: убрать запись и нажать Apply.
