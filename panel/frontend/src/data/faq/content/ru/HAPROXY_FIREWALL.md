# Firewall HAProxy

Сетевой фильтр на одном сервере: разрешает или запрещает трафик по портам, протоколам, направлению и IP.

## Параметры правила
- **Port** — номер 1–65535 или `any`
- **Protocol** — `tcp`, `udp`, `any`
- **Action** — `allow` или `deny`. По умолчанию всё запрещено: нет правила — трафик блокируется.
- **Direction** — `in` (входящий) или `out` (исходящий)
- **From IP** — источник: IP, CIDR (`10.0.0.0/24`) или `0.0.0.0/0` (все)

Типовой набор:

```
allow tcp 22 from <ваш IP>    - SSH только с вашего IP
allow tcp 80 from 0.0.0.0/0   - HTTP всем
allow tcp 443 from 0.0.0.0/0  - HTTPS всем
allow tcp 9100 from <IP панели> - мониторинг только с панели
```

## Полезно знать
- Это правила iptables/nftables конкретного сервера. Blocklist — другое: глобальные списки IP через ipset на всех нодах сразу.
- После добавления правила нажмите Apply — firewall перезагрузится.
- Порт открыт, а сервис недоступен — проверьте, что приложение слушает: `ss -tlnp | grep :443`.
- Случайно заблокировали SSH — спасёт только консоль хостинга: `iptables -F`, затем вернуть правила через панель.
