# Firewall HAProxy

Управление правилами сетевого фильтра на сервере. Позволяет разрешать/запрещать трафик по портам, протоколам, направлениям и IP.

## Что можно делать

- Включать/выключать firewall одной кнопкой
- Добавлять правила: порт, протокол, действие, направление, IP
- Удалять отдельные правила
- Видеть все активные правила в таблице

## Ключевые понятия

**Port** — номер порта (1–65535) или `any`.

**Protocol** — `tcp`, `udp`, `any`.

**Action** — `allow` (разрешить) или `deny` (запретить). **По умолчанию deny** — если нет правила, трафик блокируется.

**Direction** — `in` (входящий) или `out` (исходящий).

**From IP** — источник. Можно IP, CIDR (`10.0.0.0/24`) или `0.0.0.0/0` (все).

## Базовые правила для типичного сервера

```
allow tcp 22 from <your IP>   - SSH только с вашего IP
allow tcp 80 from 0.0.0.0/0   - HTTP всем
allow tcp 443 from 0.0.0.0/0  - HTTPS всем
allow tcp 9100 from <panel IP> - мониторинг только с панели
deny all                      - всё остальное закрыть
```

## Частые вопросы

### В чём разница с Blocklist?
**Firewall** — правила на уровне iptables/nftables для конкретного сервера, управляются через эту страницу. **Blocklist** — глобальные списки IP, применяемые через ipset на всех нодах централизованно.

Используйте firewall для индивидуальных правил сервера (какие порты открыты), Blocklist — для глобальной блокировки (кого банить вообще).

### Случайно заблокировал SSH, что делать?
Консоль хостинга → войти → `iptables -F` (или аналог) → восстановить доступ → вернуть правила через панель, включая SSH.

### allow 443 from 0.0.0.0/0 — это безопасно?
Само по себе — да, HTTPS открыт всем. Безопасность обеспечивает приложение, которое слушает на 443. Firewall тут просто говорит «пропускай трафик».

### Правило «deny all» нужно в конце?
Зависит от режима работы firewall. В большинстве реализаций по умолчанию запрет — явное «deny all» в конце не обязательно, но не вредит.

## Если что-то не работает

- Правило добавлено, но не работает → нажмите Apply, firewall должен быть перезагружен
- Порт открыт, но сервис недоступен → проверьте, что приложение реально слушает: `ss -tlnp | grep :443`
- Случайно заблокировал себя → только через консоль хостинга
