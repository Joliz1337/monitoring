# Firewall профили (UFW)

Централизованное управление правилами UFW на нодах. Один профиль — набор правил, который раскатывается на много серверов сразу. Аналог HAProxy-конфигов, но для UFW.

## Что делает
- Профиль с правилами UFW: порт, протокол, IP-источник, направление, действие
- Политики по умолчанию для входящего и исходящего трафика
- Привязка серверов: один сервер — в одном профиле
- Раскатка на все привязанные серверы одной кнопкой (Sync)
- Drift-детекция по SHA256-хешу правил: видно, кто синхронизирован, а кто отстал
- История синхронизаций с причинами ошибок

## Как настроить
- **Default incoming / outgoing** — что делать с трафиком вне правил: `deny` (молча), `reject` (с ICMP), `allow`. Обычно `deny incoming` + `allow outgoing`.
- **From IP** пусто = от любого; можно IP (`10.0.0.1`) или подсеть (`10.0.0.0/24`).
- **Sync** атомарен: `ufw reset → defaults → rules → enable`. Перед reset нода пишет снимок в `/etc/monitoring/ufw_backup_<ts>.json` и откатывается из него, если apply упал.

## Полезно знать
- Нода **откажется** применить профиль без `allow 9100/tcp in` (порт API ноды) — иначе панель потеряет связь. Обойти можно галочкой **force**, но без неё связь сохранится при ошибке.
- «Sync всех» с force опасен: ошиблись с 9100 — потеряете связь со всем пулом разом.
- Отвязка сервера рвёт связь в БД, но правила на ноде НЕ откатываются — примените другой профиль или почистите вручную.
- `comment` у правила — пометка для людей, в хеш не входит и на ноду не передаётся.
- Статусы: **synced** (хеш совпал), **pending** (в процессе или правило изменилось после sync), **failed** (нода недоступна / ошибка валидации), **rolled_back** (apply упал, откат из бэкапа).
- `failed: Allow rule for node API port 9100/tcp missing` — добавьте `allow 9100/tcp in` или включите force на свой риск.
