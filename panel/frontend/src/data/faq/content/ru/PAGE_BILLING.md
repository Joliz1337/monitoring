# Учёт ресурсов

Отслеживание использования ресурсов каждого сервера, лимитов квот и дат истечения. Удобно для подсчёта затрат и планирования бюджета, особенно при работе с облачными провайдерами (Yandex Cloud).

## Что можно делать

- Видеть текущие потреблённые часы и квоту каждого сервера
- Задавать `quota_hours` (лимит) и `expiry_date` (до какого числа действует)
- Группировать серверы в папки, перетаскивать drag-and-drop
- Автосинхронизация данных с Yandex Cloud через привязанный YC Account
- Продлевать дату истечения (Extend) или добавлять часы к квоте (TopUp)
- Смотреть историю затрат

## Ключевые понятия

**Quota** — лимит часов работы за период. Обычно месяц.

**Hours used** — сколько часов уже потрачено в текущем периоде. Считается от даты начала до сейчас.

**Expiry date** — дата, после которой текущая квота сбрасывается/перезагружается. Не путать с «удаление сервера».

**Extend** — сдвиг даты истечения вперёд без изменения квоты. Если у вас квота 100ч, использовано 80, Extend на +30 дней = всё ещё 80/100, но срок больше.

**TopUp** — добавление часов к квоте без изменения даты. Квота 100ч, use 80, TopUp +50 = стало 80/150.

**YC Account** — учётные данные Yandex Cloud для автоматической синхронизации часов. Без него ввод ручной.

**Auto-Sync** — фоновый процесс забирает из Yandex Cloud актуальные часы использования по каждому серверу.

## Частые вопросы

### Чем отличаются Extend и TopUp?
**Extend** — сдвигает срок (дату expiry), часы не трогает. **TopUp** — добавляет часы, дату не трогает. Нужно и то, и другое — делайте оба по очереди.

### Что если сервер исчерпал квоту?
В текущей реализации это **предупреждение**, сервер продолжает работать — панель просто показывает 100%. Если нужно жёстко останавливать — делайте алерт по метрике и вручную гасите сервер.

### YC Account не синхронизирует, в чём дело?
Проверьте в настройках:
1. OAuth токен действителен (Yandex Cloud периодически перевыпускает)
2. Folder ID указан корректно
3. У токена есть права `viewer` на нужный folder
4. Панель имеет сетевой доступ к `*.api.cloud.yandex.net`

### Можно ли импортировать серверы в учёт автоматически?
Да, через Yandex Cloud Account. Панель подтянет список VM и создаст записи. Для VPS вне YC — только вручную.

### Что показывает «days left»?
Количество дней до даты expiry. Красный если < 0 (просрочено), жёлтый < 14 дней, зелёный > 14.

## Если что-то не работает

- Цифры не обновляются → проверьте, что YC Account активен и токен валиден
- Сервер пропал из списка → удалён вручную или из YC; добавьте заново
- Drag-and-drop не переносит → обновите страницу, возможно старое состояние кэша
