# Внешние источники блок-листа

Импорт готовых списков IP из внешних URL. Каждый час панель обновляет списки и синхронизирует их на ноды через ipset.

## Типичные источники

**Spamhaus DROP** — список заведомо вредных сетей (ботнеты, спам-фермы).
`https://www.spamhaus.org/drop/drop.txt`

**Spamhaus EDROP** — расширенный DROP.
`https://www.spamhaus.org/drop/edrop.txt`

**FireHOL Level 1** — аггрегация из множества источников.
`https://iplists.firehol.org/files/firehol_level1.netset`

**Team Cymru BOGONS** — сети, которые никогда не должны появляться в интернете (RFC1918, reserved).
`https://www.team-cymru.org/Services/Bogons/fullbogons-ipv4.txt`

## Формат списка

Плоский текстовый файл, по одному IP или CIDR на строку:
```
1.2.3.0/24
5.6.7.8
10.20.30.0/23
```

Комментарии (строки с `#`) пропускаются. Пустые строки игнорируются.

## Параметры источника

**name** — метка для удобства (отображается в UI).

**url** — полный HTTPS URL со списком.

**direction** — блокировать `in` (входящий) или `out` (исходящий) трафик с этих IP.

**last_sync** — время последней успешной синхронизации.

**status** — `ok`, `error` (с кратким сообщением) или `pending`.

## Частые вопросы

### Как часто обновляются?
По умолчанию — раз в час. Можно принудительно кнопкой Refresh.

### Что если источник недоступен?
Статус станет `error`. Старые записи из этого источника **остаются** в ipset — панель не удаляет их, пока не получит свежий список. Это защита от «вдруг источник упал, пусть пока работает старый список».

### Источник содержит миллион IP — нода справится?
Да. ipset эффективно хранит такие объёмы (используется хэш-таблица). Даже 10M записей — норма для современного ядра.

### Добавить свой кастомный URL можно?
Да, нужен только HTTPS URL с текстовым списком в формате выше.

### Что произойдёт при удалении источника?
IP из этого источника уйдут из ipset при следующей полной синхронизации. Индивидуальные записи не удалятся — они управляются вручную.

## Если что-то не работает

- Источник `error 403` → URL требует аутентификации или rate-limited
- Источник `error 404` → URL устарел, найдите актуальный
- Статус OK, но блокировка не работает → проверьте direction (in/out) и что ipset реально на ноде (`ipset list`)
