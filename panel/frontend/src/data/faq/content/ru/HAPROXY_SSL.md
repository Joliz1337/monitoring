# SSL-сертификаты HAProxy

Управление SSL/TLS сертификатами на одном сервере: выпуск через Let's Encrypt, ручная загрузка, автообновление, удаление.

## Что можно делать

- Генерировать новые сертификаты через Let's Encrypt (ACME)
- Загружать собственные сертификаты (cert + private key)
- Обновлять сертификаты (Renew)
- Включать автообновление
- Видеть детали: subject, issuer, SANs, дата истечения

## Ключевые понятия

**Subject** — кому выдан сертификат (обычно домен).

**Issuer** — кем выдан. Для Let's Encrypt это `Let's Encrypt Authority X3` или аналог.

**SANs (Subject Alternative Names)** — дополнительные домены, которые покрывает тот же сертификат. Например, `example.com`, `www.example.com`, `api.example.com`.

**Expiration date** — дата истечения. Let's Encrypt выдаёт на 90 дней.

**ACME challenge** — способ доказать владение доменом. Для HAProxy обычно HTTP-challenge (файл в `/.well-known/acme-challenge/`).

**fullchain.pem + privkey.pem** — два файла, из которых состоит SSL-сертификат. HAProxy их объединяет в один `.pem`.

## Частые вопросы

### Как сгенерировать сертификат?
1. Убедитесь, что домен направлен на этот сервер (A/AAAA запись)
2. Порт 80 открыт (для HTTP challenge)
3. Нажмите «Generate», укажите email и домен
4. Панель выполнит ACME challenge и сохранит сертификат
5. Добавьте путь к сертификату в правило HAProxy, перезагрузите

### Можно ли загрузить свой сертификат (купленный/самоподписанный)?
Да. «Upload» → выберите файл `.pem`, содержащий `fullchain + privkey`, или два файла отдельно. Панель объединит их в нужный HAProxy формат.

### Что такое «expiring soon»?
Меньше 30 дней до истечения. Нужно обновить, иначе клиенты увидят ошибку SSL.

### Почему Let's Encrypt не работает?
Самые частые причины:
1. Домен не указывает на этот сервер (A-запись)
2. Порт 80 закрыт firewall
3. Уже стоит HAProxy/nginx на 80, HAProxy не может поднять ACME listener
4. Rate limit Let's Encrypt (5 попыток в час на домен)

### Чем отличается от Wildcard SSL на отдельной странице?
Wildcard SSL — централизованное управление wildcard-сертификатом для множества серверов через DNS challenge (Cloudflare). Здесь — обычные сертификаты на один сервер через HTTP challenge.

## Если что-то не работает

- Generate падает → проверьте DNS, порт 80, rate limits
- Сертификат загружен, но HAProxy не использует → добавили в правило? reload делали?
- Auto-renew не работает → в логах обычно ошибка конкретная (чаще всего те же причины, что при manual)
