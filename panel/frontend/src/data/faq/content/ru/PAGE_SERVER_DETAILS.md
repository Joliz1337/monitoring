# Детали сервера

Глубокая диагностика одного сервера в реальном времени. Сюда заходят, когда надо понять, почему именно этот сервер тормозит.

## Что показывает
- CPU, RAM, диск и сеть с графиками за 1ч / 6ч / 24ч
- CPU по каждому ядру отдельно — видно, забито одно ядро или нагрузка размазана
- Таблицу процессов с сортировкой по CPU или RAM
- Терминал для команд, кнопки Reboot / Shutdown, примечания (локально в браузере)

## Полезно знать
- Одно ядро в 100%, остальные простаивают — приложение не параллелится, добавление ядер не поможет.
- Load Average больше числа ядер — система перегружена, процессы ждут очереди. Короткие пики — норма.
- Убить процесс: `kill <PID>` (мягко) или `kill -9 <PID>` (жёстко), PID виден в таблице процессов.
- **Reboot** вернёт сервер за 1–2 минуты. **Shutdown** на VPS без физического доступа = «не поднимется без панели хостинга».
- Нет температур — датчики недоступны (частая ситуация на VPS/контейнерах).
- Метрики пустые — агент не отвечает, на странице «Серверы» запустите «Тест».
