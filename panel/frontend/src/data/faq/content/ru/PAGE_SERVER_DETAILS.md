# Детали сервера

Глубокая диагностика одного сервера в реальном времени: CPU по ядрам, RAM, диск, сеть, процессы, терминал, примечания. Сюда заходят, когда надо понять, почему именно этот сервер тормозит.

## Что можно делать

- Смотреть CPU, RAM, диск, сеть с графиками за 1ч / 6ч / 24ч
- Разворачивать CPU по каждому ядру — видно, равномерно распределена нагрузка или одно ядро забито
- Открыть таблицу процессов, отсортировать по CPU или RAM, найти «съедатель»
- Выполнить команду на сервере через встроенный терминал
- Перезагрузить или выключить сервер
- Оставить примечания по серверу (локально, на вкладке справа)

## Ключевые понятия

**Per-core CPU** — использование каждого ядра отдельно. Важно: если одно ядро в 100%, а остальные в 5% — у вас приложение не умеет параллелиться, и добавление ядер не поможет.

**Load Average** — очередь на CPU. Если 1-мин LA больше числа ядер — процессы ждут, система перегружена. Короткие скачки — нормально, устойчиво высокое значение — проблема.

**Процесс Zombie** — процесс завершился, но родитель не вызвал `wait()` и не «похоронил» его. Сам по себе не ест ресурсы, но указывает на баг в коде родителя.

**Shell timeout (терминал)** — максимальное время выполнения команды. По истечении процесс получит SIGTERM, потом SIGKILL. По умолчанию 30 сек — достаточно для большинства команд.

**Disk I/O** — скорость чтения/записи на диск, не путать с сетевым RX/TX. Если высокий I/O и низкий CPU — узкое место в диске.

## Частые вопросы

### Load Average 8 при 4 ядрах — это катастрофа?
Это перегрузка вдвое. Нужно: открыть процессы, отсортировать по CPU, найти топ-3 прожорливых и решить — оптимизировать, ограничить, или добавить ресурсов. Короткие пики до 2×cores — нормально; устойчиво высокое — проблема.

### Как убить процесс?
Через терминал: `kill <PID>` (мягко, SIGTERM) или `kill -9 <PID>` (жёстко, SIGKILL). PID виден в таблице процессов.

### Чем Reboot отличается от Shutdown?
**Reboot** — перезагрузка, сервер вернётся в онлайн через 1–2 минуты. **Shutdown** — выключение до ручного включения (или `wake-on-lan`). Для VPS без физического доступа Shutdown может значить «сервер не поднимется, пока не зайдёте в панель хостинга».

### Почему температуры нет?
Датчики CPU не всегда доступны (особенно на VPS/контейнерах). На физическом железе — проверьте `sensors` / `lm-sensors` пакет на ноде.

### Где посмотреть трафик?
Отдельная страница «Трафик» — кнопка на этой же странице. Там графики сети по периодам, разбивка TCP-состояний, порты.

## Если что-то не работает

- Метрики пустые → агент не отвечает, вернитесь на страницу «Серверы» и запустите «Тест»
- Терминал зависает → увеличьте timeout перед запуском, либо команда требует интерактивного ввода (такие не поддерживаются)
- Per-core CPU не показывается → возможно, данные ещё не собрались, подождите один интервал обновления
