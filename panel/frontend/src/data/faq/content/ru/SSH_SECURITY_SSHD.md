# Конфигурация sshd

Параметры демона SSH на ноде. Правите здесь — панель применяет на сервере и перезапускает sshd.

## Ключевые параметры

**Port** — номер порта SSH. Стандартно 22. Изменение порта снижает фоновый скан-трафик, но не заменяет нормальную защиту.

**PermitRootLogin** — разрешение входа под root:
- `yes` — можно по паролю или ключу (опасно)
- `no` — нельзя совсем
- `prohibit-password` — только по ключу, пароль отбрасывается (рекомендуется)

**PasswordAuthentication** — разрешён ли вход по паролю. `yes` — можно, `no` — только ключи.

**PubkeyAuthentication** — разрешены ли ключи. Почти всегда `yes`.

**MaxAuthTries** — сколько раз можно ввести пароль/ключ за одно соединение. По умолчанию 6. Рекомендую 3.

**LoginGraceTime** — сколько секунд у клиента есть на авторизацию. По умолчанию 120. Рекомендую 30.

**ClientAliveInterval** / **ClientAliveCountMax** — периодический keep-alive, закрывает «зависшие» сессии.

**MaxSessions** — сколько одновременных каналов в одном соединении (multiplex). 10 — разумно.

**X11Forwarding** — проброс X-сервера. На production-серверах не нужен, ставьте `no`.

## Рекомендуемый минимум

- PermitRootLogin `prohibit-password`
- PasswordAuthentication `no` (если у вас настроены ключи!)
- MaxAuthTries `3`
- LoginGraceTime `30`
- X11Forwarding `no`

## Частые вопросы

### Смена порта — нужна ли?
Это «security through obscurity», не настоящая защита. Плюс: меньше спама в логах от фоновых сканеров. Минус: ваши клиенты должны указывать `-p` или `Port` в `~/.ssh/config`. Для удобства оставьте 22 и положитесь на Fail2ban + ключи.

### Отключил пароль — теперь не могу войти
Потому что ключ не был загружен заранее. Вариант: войти через консоль хостинга, включить `PasswordAuthentication yes` обратно, добавить ключ, снова выключить.

### Что будет, если поставить PermitRootLogin no, а у меня только root?
Заблокируете себя. Перед этим создайте обычного пользователя, дайте ему sudo, проверьте вход, и только потом закрывайте root.

## Если что-то не работает

- После применения не могу войти → консоль хостинга, откатить настройки вручную: `/etc/ssh/sshd_config`
- Не применяется → проверьте связь с нодой и логи агента
- «sshd: bad configuration option» → опечатка в одном из параметров, панель покажет ошибку
