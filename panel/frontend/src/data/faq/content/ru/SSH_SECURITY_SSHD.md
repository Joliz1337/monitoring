# Конфигурация sshd

Параметры демона SSH на ноде. Правите здесь — панель применяет их на сервере и перезапускает sshd.

## Параметры
- **Port** — порт SSH, по умолчанию 22. Смена снижает спам сканеров в логах, но не заменяет защиту.
- **PermitRootLogin** — вход под root: `prohibit-password` (только ключ, рекомендуется), `no`, `yes` (опасно).
- **PasswordAuthentication** — вход по паролю. `no` — только ключи.
- **PubkeyAuthentication** — вход по ключу. Почти всегда `yes`.
- **MaxAuthTries** — попыток за соединение. По умолчанию 6, рекомендуется 3.
- **LoginGraceTime** — секунд на авторизацию. По умолчанию 120, рекомендуется 30.
- **ClientAliveInterval / ClientAliveCountMax** — keep-alive, закрывает зависшие сессии.
- **MaxSessions** — каналов в одном соединении. 10 — норма.
- **X11Forwarding** — проброс X-сервера. На сервере не нужен, ставьте `no`.

## Полезно знать
- Перед `PasswordAuthentication no` загрузите ключ заранее, иначе войти будет нечем — спасёт только консоль хостинга.
- Перед `PermitRootLogin no` создайте обычного пользователя с sudo и проверьте вход, иначе заблокируете себя.
- Конфиг лежит в `/etc/ssh/sshd_config`; откатить вручную можно через консоль хостинга.
