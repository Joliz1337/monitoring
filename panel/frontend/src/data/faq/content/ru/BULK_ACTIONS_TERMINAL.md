# Массовый терминал

Выполнение shell-команды на множестве серверов одновременно. Быстрее, чем заходить по SSH в каждый.

## Что можно делать

- Вводить одну команду или многострочный скрипт
- Выбирать timeout (30 сек — 10 мин)
- Выбирать shell: `sh` (максимальная совместимость) или `bash` (расширения, массивы, etc.)
- Видеть результат по каждому серверу: exit code, stdout, stderr
- Раскрывать полный вывод кликом

## Ключевые понятия

**Exec mode** — одна команда, выполняется напрямую в shell.

**Script mode** — многострочный блок, сохраняется во временный файл и выполняется как скрипт. Поддерживает переменные, функции, циклы.

**Timeout** — лимит времени на выполнение. По истечении процесс получает SIGTERM, потом SIGKILL. Важно ставить адекватный — 30 сек для простых команд, больше для длинных операций.

**Exit code** — код возврата команды. `0` = успех, любое другое число = ошибка.

## Типичные команды

```bash
# Версии ОС на всех нодах
lsb_release -a

# Дата и время
date

# Свободное место
df -h

# Память
free -m

# Установленный пакет
dpkg -l | grep nginx

# Последние строки лога
tail -20 /var/log/syslog
```

## Частые вопросы

### `sudo` нужен?
Агент уже работает от root, `sudo` не нужен. Он даже может мешать, если не настроен. Просто пишите команду.

### Интерактивные команды работают?
Нет. Нельзя использовать `htop`, `vim`, `nano`, `passwd` и другие, которые требуют ввода. Только неинтерактивные.

### Как выполнить цепочку команд?
Через `&&` (выполнять только если предыдущая успешна) или `;` (выполнять независимо):
```bash
apt update && apt install -y curl
systemctl stop nginx; systemctl start nginx
```

### Команда работает на 1 сервере, а на 5 — падает по timeout
Увеличьте timeout или оптимизируйте команду. Возможно, на тех серверах медленный диск/сеть/нагрузка.

### Опасно — можно сломать всё разом?
Да. Внимательно проверяйте команду перед запуском, особенно если в ней `rm`, `iptables`, `systemctl stop`, `reboot`. Ошибка повторится 50 раз.

## Если что-то не работает

- Все серверы вернули timeout → у вас неверная команда или очень медленная
- Часть серверов «pending» → повторите; агент мог не ответить
- stdout пустой, exit 0 → команда выполнилась, просто ничего не вывела (это ок)
