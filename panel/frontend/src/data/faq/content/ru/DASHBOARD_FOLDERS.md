# Папки на дашборде

Способ организовать десятки/сотни серверов в логические группы. Помогает быстро находить нужные ноды и наводить порядок.

## Что можно делать

- Создать папку (например, `Production VPN`, `Staging`, `Dev`)
- Перетащить сервер мышью в папку
- Переименовать папку
- Свернуть/развернуть папку (экономит место на экране)
- Удалить папку (серверы не удалятся, просто выпадут из неё)
- Менять порядок папок перетаскиванием

## Как это хранится

Состояние папок (список, порядок, свёрнутость) хранится **локально в браузере**, в localStorage. Это значит:
- У каждого администратора своя структура
- Чистка кэша = всё сбросится
- Нельзя «синхронизировать» папки между устройствами

## Частые вопросы

### Сервер пропал из папки после перезагрузки — почему?
Возможно, сбросился localStorage. Перетащите его обратно. Или браузер в режиме приватного просмотра, который не сохраняет состояние.

### Можно ли сделать подпапки?
В текущей реализации — нет, только один уровень. Если нужна иерархия — используйте названия вроде `EU-Production-VPN`.

### Сервер может быть в нескольких папках?
Нет, только в одной. Это сознательное решение, иначе пользователь может забыть, где что.

### Как переместить несколько серверов сразу?
По одному drag-and-drop. Массового переноса в текущей версии нет.

## Если что-то не работает

- Drag-and-drop не работает → проверьте, что браузер современный (Chrome/Firefox последних версий)
- Папка не создаётся → вероятно, localStorage заполнен или отключён (private mode)
- Состояние сбрасывается при каждом заходе → браузер чистит localStorage на выходе
