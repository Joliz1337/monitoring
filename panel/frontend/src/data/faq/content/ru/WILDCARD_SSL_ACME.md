# ACME + Cloudflare API

Автоматическое создание wildcard-сертификатов через Let's Encrypt с DNS challenge через Cloudflare API.

## Как это работает

1. Вы вводите домен (`example.com`) и нажимаете Issue
2. Панель идёт в Let's Encrypt, просит сертификат на `*.example.com`
3. Let's Encrypt говорит: «докажи владение доменом, создай TXT-запись `_acme-challenge.example.com` = `xyz`»
4. Панель через Cloudflare API создаёт эту запись в вашей DNS-зоне
5. Let's Encrypt проверяет, видит TXT, выдаёт сертификат
6. Панель удаляет временную TXT-запись
7. Сертификат сохраняется в БД панели

## Что нужно для работы

**Cloudflare API Token** — с правами:
- `Zone.Zone.Read` (на нужную зону)
- `Zone.DNS.Edit` (на нужную зону)

Создаётся на https://dash.cloudflare.com/profile/api-tokens.

**Email** — для регистрации аккаунта ACME (Let's Encrypt). Не публикуется, используется только для уведомлений о скором истечении.

**Renew Days Before** — за сколько дней до истечения автоматически обновлять. По умолчанию 30.

## Почему именно DNS challenge

Для wildcard-сертификатов **только DNS challenge** работает. HTTP challenge не поддерживает wildcard, потому что нельзя доказать владение «всеми поддоменами» через файл на одном сервере.

DNS challenge доказывает, что вы управляете **зоной**, значит можете выпускать сертификаты для любого поддомена этой зоны.

## Частые вопросы

### Зачем нужен Cloudflare именно?
Технически можно любого провайдера с API. Cloudflare выбран потому что:
- Бесплатный API
- Простая аутентификация токеном
- Широко используется
- Хорошо документирован

Для других провайдеров нужен отдельный код интеграции, которого здесь нет.

### Мой DNS не в Cloudflare
Два варианта:
1. Перенести DNS на Cloudflare (бесплатно, 10 минут)
2. Использовать другой способ получения сертификата (например, certbot локально) и загрузить сертификат как «свой»

### Rate limits Let's Encrypt?
**20 сертификатов на зарегистрированный домен в неделю**. **5 duplicate certs в неделю**. **300 заказов в 3 часа**. Для нормального использования — не проблема. Если упрётесь в лимиты, используйте staging окружение для тестов (отдельный эндпоинт без лимитов, но сертификаты не доверяются браузерами).

### Куда идут ключи сертификата?
- fullchain.pem + privkey.pem хранятся в БД панели
- При деплое копируются на каждый настроенный сервер по SSH через агент
- Приватный ключ никогда не покидает панель и агенты

## Если что-то не работает

- `invalid token` → Cloudflare Token неверный или истёк, создайте новый
- `zone not found` → токен не имеет доступа к этой зоне
- `timeout waiting for DNS` → CF медленно распространяет запись, бывает, повторите
- `rate limit` → ждите или staging окружение
