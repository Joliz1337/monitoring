# Отслеживание портов

Добавление конкретных портов в мониторинг трафика, чтобы видеть, сколько данных прошло через каждый за период.

## Как настроить
- Добавьте порт в список — через минуту-две появится RX/TX и суммарный трафик за 30 дней.
- Отслеживать можно несколько портов сразу, лишние — удалять.
- Частые порты: `22` SSH, `25` SMTP, `53` DNS, `80` HTTP, `443` HTTPS, `3306` MySQL, `5432` PostgreSQL, `6379` Redis, `27017` MongoDB.

## Полезно знать
- Под капотом — правила iptables/nftables со счётчиком на порт. Правила прозрачны (ACCEPT), на трафик не влияют, нагрузка практически нулевая.
- Трафик `0` байт — на порту реально нет данных, приложение слушает на другом интерфейсе, или правило ещё не применилось. Проверка: `ss -tln | grep :443`.
- Диапазоны портов не поддерживаются — только конкретные.
- Цифры обнулились — сервер перезагружался, счётчики сбросились.
- Расхождение с `iftop`/`nethogs` — разные методы измерения, обычно в пределах погрешности.
