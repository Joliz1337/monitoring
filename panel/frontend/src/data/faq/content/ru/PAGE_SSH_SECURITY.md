# SSH Security

Управление безопасностью SSH сразу на всех нодах: параметры sshd, Fail2ban, ключи и баны в одном месте.

## Что делает
- Правит параметры sshd: порт, root-доступ, пароль/ключи, таймауты, `MaxAuthTries`
- Включает Fail2ban и настраивает `max_retry` / `find_time` / `ban_time`
- Показывает забаненные IP и разбанивает руками
- Загружает публичные ключи для входа без пароля
- Применяет пресеты «Recommended» и «Maximum» массово на выбранные ноды

## Полезно знать
- `PermitRootLogin`: `prohibit-password` — root только по ключу (рекомендуется), `no` — root запрещён, `yes` — опасно.
- Пресет «Maximum» отключает вход по паролю — применяйте, только если уже проверили вход по ключу, иначе закроете себе доступ.
- Меняя порт, сначала откройте новый в firewall и проверьте вход с другого терминала, не закрывая текущий.
- Забанили себя Fail2ban — зайдите через консоль хостинга (VNC) и сделайте `fail2ban-client unban <ваш IP>`, а свой IP добавьте в `ignoreip`.
