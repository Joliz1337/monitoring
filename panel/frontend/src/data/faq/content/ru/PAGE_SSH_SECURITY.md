# SSH Security

Централизованное управление безопасностью SSH и защитой от brute-force на всех нодах. Позволяет настроить sshd, включить Fail2ban, раскидать ключи, управлять банами.

## Что можно делать

- Править параметры SSH: порт, разрешение root, пароль/ключи, таймауты, max_auth_tries
- Включать/выключать Fail2ban, настраивать max_retry / ban_time / find_time
- Смотреть список заблокированных Fail2ban IP и разбанивать руками
- Загружать публичные SSH-ключи для авторизации без пароля
- Применять готовые пресеты безопасности («Recommended», «Maximum»)
- Массово применять текущую конфигурацию ко всем выбранным серверам

## Ключевые понятия

**sshd** — демон SSH на ноде, читает `/etc/ssh/sshd_config`. Любое изменение требует перезапуска сервиса.

**PermitRootLogin** — разрешён ли вход под root. Значения: `yes` (разрешено, опасно), `no` (запрещено), `prohibit-password` (только по ключу, рекомендуется).

**Password / Pubkey authentication** — авторизация паролем или публичным ключом. Ключ всегда безопаснее: его нельзя «перебрать» за разумное время.

**Fail2ban** — демон, который читает `auth.log`, считает неудачные попытки SSH и банит IP через firewall. Правило в Fail2ban называется **jail**.

**max_retry** — сколько неудачных попыток войти до бана. **find_time** — окно, в котором эти попытки считаются. **ban_time** — длительность бана. Пример: `max_retry=5, find_time=10m, ban_time=1h` → «5 неудачных за 10 минут → бан на час».

**Jail** — конкретное правило в Fail2ban. Для SSH это `sshd` jail, для nginx есть свои, но здесь мы правим только SSH.

## Частые вопросы

### Безопасно ли менять порт SSH?
Да, но осторожно:
1. Не закрывайте текущее соединение, пока не проверите новый порт
2. Сначала разрешите новый порт в firewall, потом меняйте sshd
3. Проверьте вход с другого терминала, прежде чем закрыть старый

### Сам себя забанил Fail2ban, что делать?
Войдите через панель хостинга (VNC/console), запустите `fail2ban-client unban <ваш IP>` или временно отключите Fail2ban. На будущее — добавьте свой IP в whitelist (`ignoreip`).

### PermitRootLogin — что выбрать для prod?
**prohibit-password** — золотой стандарт: root можно, но только по ключу, пароль не примут. Или **no** — root вообще нельзя, заходите под обычным пользователем и делаете `sudo`.

### Как загрузить SSH-ключ?
1. На вашей машине: `cat ~/.ssh/id_ed25519.pub` (или `id_rsa.pub`)
2. Скопируйте строку (`ssh-ed25519 ... user@host`)
3. В панели вставьте в поле Public Key, задайте имя, сохраните
4. На сервере ключ добавится в `~/.ssh/authorized_keys`

### Что делает пресет «Maximum»?
Отключает вход по паролю (только ключи), запрещает root, снижает max_auth_tries, уменьшает login grace time, включает Fail2ban со строгими параметрами. Применять только если у вас есть рабочий ключ и вы проверили вход!

## Если что-то не работает

- После смены порта не могу войти → проверьте firewall и `sshd_config` через консоль хостинга
- Fail2ban не банит → убедитесь, что сервис запущен (`systemctl status fail2ban`) и jail включён
- SSH-ключ не работает → проверьте права: `~/.ssh/` должно быть `700`, `authorized_keys` — `600`
- Забанил самого себя → консоль хостинга, разбан, whitelist
