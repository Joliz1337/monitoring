# Fail2ban

Защита от brute-force атак на SSH. Читает `auth.log`, считает неудачные попытки и временно банит IP через firewall.

## Ключевые параметры

**enabled** — включён ли Fail2ban вообще.

**max_retry** — сколько неудачных попыток входа за `find_time` до бана. Стандартно 5.

**find_time** — окно (в секундах) для подсчёта попыток. Стандартно 600 секунд (10 минут).

**ban_time** — на сколько секунд IP блокируется. Стандартно 600 (10 минут), рекомендую 3600+ (час).

## Как это работает

1. Кто-то пытается войти с IP `1.2.3.4`
2. Попытка фиксируется в `/var/log/auth.log`
3. Fail2ban читает лог и считает: сколько `Failed password` от `1.2.3.4` за `find_time`
4. Если больше `max_retry` → добавляет IP в iptables/nftables с правилом DROP на `ban_time`
5. По истечении `ban_time` правило удаляется, IP снова может пытаться

## Рекомендуемые значения

| Для | max_retry | find_time | ban_time |
|-----|-----------|-----------|----------|
| Личный VPS | 5 | 600 | 3600 |
| Публичный сервис | 3 | 300 | 86400 |
| Максимальная защита | 2 | 600 | 604800 |

## Частые вопросы

### Забанил сам себя, как разбанить?
На сервере: `fail2ban-client unban <ваш IP>`. Или через панель — список забаненных IP с кнопкой unban.

### Как не банить свои IP (whitelist)?
Добавьте свои IP в `ignoreip` в конфиге Fail2ban. В панели это отдельное поле или нужно ручками править на сервере.

### Fail2ban vs sshd MaxAuthTries — в чём разница?
**MaxAuthTries** — сколько попыток в одном подключении (6 паролей, потом разрыв, можно сразу переподключиться). **Fail2ban** — сколько **подключений** с неудачами за окно времени (5 неудачных подключений → бан на час, вообще нельзя подключаться).

### Ban time слишком длинный — плохо?
Для атакующего — хорошо. Для вас — плохо, если случайно забанитесь. Разумный компромисс: 1–24 часа + whitelist.

## Если что-то не работает

- Fail2ban «enabled», но никого не банит → проверьте, что сервис запущен: `systemctl status fail2ban`
- Не читает логи → может быть, путь к `auth.log` изменён (в journald), нужна настройка backend
- Разблокировка не срабатывает → смотрите `fail2ban.log`, иногда бан висит из-за битого состояния
