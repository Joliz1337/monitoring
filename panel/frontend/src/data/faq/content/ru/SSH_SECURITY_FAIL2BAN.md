# Fail2ban

Защита от brute-force на SSH. Читает `/var/log/auth.log`, считает неудачные попытки входа и временно банит IP через iptables/nftables.

## Параметры
- **enabled** — включён ли Fail2ban.
- **max_retry** — неудачных попыток за `find_time` до бана. По умолчанию 5.
- **find_time** — окно подсчёта попыток в секундах. По умолчанию 600 (10 минут).
- **ban_time** — длительность бана в секундах. По умолчанию 600, рекомендуется 3600+ (час).

## Полезно знать
- Отличие от sshd `MaxAuthTries`: тот считает попытки в одном подключении, Fail2ban — число неудачных подключений за окно времени и блокирует IP целиком.
- Забанили себя — `fail2ban-client unban <ваш IP>` на сервере или кнопка unban в панели. Свои IP внесите в `ignoreip`.
- Слишком длинный `ban_time` бьёт по атакующему, но и по вам при случайном бане — разумно 1–24 часа плюс whitelist.
- Не банит — проверьте, что сервис запущен: `systemctl status fail2ban`.
