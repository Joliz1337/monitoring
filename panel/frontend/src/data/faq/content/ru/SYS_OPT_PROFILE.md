# Профиль оптимизаций: VPN или Panel

Два готовых набора системных параметров под разную нагрузку. Правило выбора одно: **через что идёт больше всего трафика — то VPN-нода**.

## Профиль VPN
Для серверов с высоким трафиком: прокси, VPN, Xray, WireGuard, OpenVPN.
- `fs.file-max` = 2M, `nf_conntrack_max` = 2M
- Socket buffers 128 MB, лимиты `nofile` и `nproc` = 1M

Тысячи соединений требуют буфер и запись в conntrack — без этого сервер упирается в лимиты ядра раньше реального потолка.

## Профиль Panel
Для лёгкой нагрузки: панели, мониторинг, Remnawave, API.
- `fs.file-max` = 524k, `nf_conntrack_max` = 262k
- Socket buffers 16 MB, `nofile` = 65k, `nproc` = 32k

Большие буферы тут только зря тратят RAM. Сомневаетесь — берите Panel.

## Полезно знать
- `nf_conntrack` — таблица соединений ядра, каждое TCP/UDP-соединение это запись; дефолтного лимита для VPN мало.
- Profile `panel` на VPN-ноде ведёт к сбросам новых соединений; симптом — `NF_CONNTRACK_OVERFLOW` в `dmesg`, срочно переставьте на vpn.
- Смена без перезагрузки частична: sysctl применяется сразу, `limits.conf` — только для новых сессий; для полного эффекта рестартуйте сервисы.
