# Серверы

Список всех нод, которые мониторит панель. Здесь добавляют новые серверы, тестируют связь и переводят старые ноды на общий ключ. Канал panel↔node зашифрован по mTLS.

## Как добавить сервер
1. **Серверы** → **Добавить сервер** — вверху формы появится **Installer Token**, скопируйте его.
2. Заполните имя, host (IP/домен), порт (по умолчанию `9100`) → **Создать**.
3. На ноде: `mon → Install Node`, вставьте токен → Enter.
4. В панели нажмите **Тест** — должно стать зелёным «Онлайн».

## Полезно знать
- **Installer Token** — один на все ноды, не одноразовый и не меняется. Если форма закрыта — откройте «Добавить сервер» снова, токен тот же.
- Порт `9100` — там идёт mTLS handshake. Открывайте его в firewall ноды только для **Panel IP**, иначе будет лишний шум в логах от сканеров.
- Бейдж **Old key** — нода на старом per-server сертификате, переведите кнопкой «Мигрировать» или баннером миграции (для legacy на `X-API-Key` нужна ручная переустановка).
- Тест красный — ищите снизу вверх: пинг → `nc -zv <ip> 9100` → `openssl s_client`. `certificate required` — норма, `unknown ca` — переустановите ноду с актуальным токеном.
- Удаление сервера стирает в панели его историю и правила, но саму ноду не трогает — docker compose на ней продолжит работать.
