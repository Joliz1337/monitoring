# Health checks

HAProxy периодически проверяет каждый backend. Несколько провалов подряд — сервер помечается DOWN и убирается из ротации; снова отвечает — UP и возвращается.

## Параметры
- **TCP check** — открыл соединение к `ip:port`? Просто и легко, но ловит только «порт упал».
- **HTTP check** — `GET /health` вернул 200? Точнее: ловит «порт открыт, но приложение висит». Нужен эндпоинт на backend.
- **HTTPS check** — то же поверх SSL, для backend на HTTPS.
- **inter** — интервал между проверками (по умолчанию 2 с)
- **rise** — сколько успехов подряд для UP (обычно 2)
- **fall** — сколько провалов подряд для DOWN (обычно 3)
- **timeout check** — сколько ждать ответа

## Полезно знать
- HTTP-check надёжнее TCP: сделайте `/health`, который проверяет БД и Redis и отдаёт 200, только если всё ок.
- Не ставьте `rise=1` / `fall=1`: при нестабильной сети сервер будет дёргаться в ротации (флаппинг). Флаппит — увеличьте `rise` и `fall`.
- DOWN при ответе 200 — частые причины: проверка идёт на другой порт, `/health` отдаёт не 200 (403/302), timeout меньше реального ответа, или firewall режет трафик до backend.
