# Блокировщик торрентов

Автоматическая блокировка IP, которые используют P2P/торренты. Работает в связке с Remnawave: оттуда получаются отчёты о торрент-активности, а панель банит эти IP через ipset на всех нодах.

## Что можно делать

- Включать/выключать автоблокировку
- Настраивать интервал опроса Remnawave (сколько ждать между циклами)
- Задавать длительность бана IP (минуты или часы)
- Исключать конкретные серверы из блокировки
- Видеть статус воркера: когда последний раз работал, успешно ли
- Смотреть статистику: сколько IP забанено всего, сколько в последнем цикле
- Запускать принудительный цикл «Poll Now»

## Ключевые понятия

**Poll Interval** — как часто воркер опрашивает Remnawave. По умолчанию 5 минут. Меньше = быстрее реакция, но больше нагрузка.

**Ban Duration** — сколько времени IP остаётся в бане. По истечении автоматически освобождается. По умолчанию 30 минут.

**Worker** — фоновый процесс панели, который:
1. Опрашивает Remnawave
2. Извлекает IP с торрент-активностью
3. Отправляет их на все ноды через ipset
4. Обрезает обработанные отчёты на Remnawave

**Excluded servers** — серверы, на которые бан не применяется. Полезно для dev-серверов или легальных раздач.

**Last poll status** — результат последнего цикла. Значения:
- `success` — всё хорошо, IP забанены
- `no_reports` — в Remnawave не было торрент-активности
- `error` — что-то сломалось, смотрите детали

## Частые вопросы

### Зачем нужна эта страница, если есть Remnawave?
Remnawave только детектирует, но не блокирует. Эта фича — мост: берёт отчёты и превращает их в фактическую блокировку через ipset на нодах.

### Почему интервал 5 минут, а не сразу?
Чтобы не создавать шквал запросов к Remnawave API. 5 минут достаточно для реакции на большинство случаев. Минимум — 1 минута.

### Чем отличается от Blocklist?
**Blocklist** — ручные глобальные списки + внешние источники. **TorrentBlocker** — автоматическая временная блокировка на основе отчётов Remnawave. Оба работают через ipset, не конфликтуют (отдельные списки с разными TTL).

### Excluded servers — зачем?
Например, dev-сервер, где вы тестируете P2P-софт. Или нода, которая сама раздаёт легальный контент. Добавляете — бан на этот сервер не применяется.

### Что значит `no_reports`?
Воркер проверил Remnawave, там не было новых отчётов о торрентах. Это хорошо — значит всё спокойно.

## Если что-то не работает

- Worker в `error` → скорее всего Remnawave недоступен или API-токен неверный. Смотрите Remnawave-страницу
- IP банятся, но всё равно подключаются → проверьте, что нода реально применяет ipset (`ipset list`)
- Слишком агрессивные баны → увеличьте `ban_duration` для менее частых срабатываний
